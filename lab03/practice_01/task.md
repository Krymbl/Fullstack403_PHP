Практика: простой router на массиве
---

В основе любого php-фреймворка лежит механизм маршрутизации,
то есть определения, какой обработчик надо вызывать в зависимости от пути,
переданного в адресной строке и HTTP-метода (иногда ещё и аргументов).
Самый просто механизм реализовать маршрутизацию
без фреймворка - создать справочник "маршрут" => "обработчик",
для чего вполне подойдут массивы.

---
Задание: создать массив "путь-обработчик"
(не полный url, а часть после доменного имени без аргументов),
где обработчики - анонимные функции (можно просто выводить в них разный текст).
Ипользуя этот массив, обработать входящий запрос и выполнить нужный обработчик.
Если обработчика для указанного пути нет, установить http статус 404 и вывести соответствующее сообщение.

Запуск встроенного сервера с указанием скрипта маршрутизации:
php -S localhost:8000 index.php При этом на какой бы маршрут внутри сервера не пытался зайти пользователь,
обработчиком всё равно будет index.php. Полезная функция: parse_url, позволяет получить нужную часть полного url
